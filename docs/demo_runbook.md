# Demo Runbook

Status: bounded local demo path for the current HackTUI POC.

## Required env vars

Required:
- `HACKTUI_DB_PASS`

Optional:
- `HACKTUI_DB_NAME` if you want to override the default current-schema local DB

Default DB-backed demo database if `HACKTUI_DB_NAME` is not set:
- `hacktui_qualification_test`

## Startup order

1. source credentials
- `set -a && source .env && set +a`

2. run the main demo command
- `mix demo.investigate case-1`

The command itself will:
- ensure the demo DB exists
- migrate it if needed
- seed deterministic case-1 data
- enable the bounded Jido runtime path
- start the required apps
- run the investigation flow
- print the bounded terminal view
- save deterministic artifacts
- create a simulated approval-required action

## Main demo command

- `mix demo.investigate case-1`

## TUI / terminal launch command

Current honest terminal launcher:
- `mix demo.view case-1`

Important
- this is a bounded terminal report view, not a full-screen interactive TUI

## Approval interaction command

- `mix demo.approve case-1`

This approves the simulated action generated by the main demo path.
The action is explicitly labeled simulated.

## Live presentation script

1. Source credentials
- `set -a && source .env && set +a`

2. Explain the current shape
- BEAM-native runtime with explicit store/hub/tui/collab/agent boundaries
- bounded Jido orchestration only for investigation
- no production-readiness claim

3. Run the main demo
- `mix demo.investigate case-1`

4. Call out what is visible
- case id
- investigation status
- matched alert ids
- shared indicators
- summary
- recommendation
- health snapshot
- Slack preview
- saved artifacts

5. Show the simulated approval path
- `mix demo.approve case-1`

6. If you want to re-show the terminal view
- `mix demo.view case-1`

## Fallback demo script

Use this if you only want the terminal view and approval interaction without relying on any Slack-facing preview discussion.

1. `set -a && source .env && set +a`
2. `mix demo.view case-1`
3. `mix demo.approve case-1`
